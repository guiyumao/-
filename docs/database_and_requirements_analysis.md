# 实验室设备与耗材管理系统数据库与需求分析

## 1. 系统概述

实验室设备与耗材管理系统面向高校实验室日常管理场景，主要解决设备、耗材、危化品、库存、借还、维修、校准、提醒、报表和权限控制等业务的统一管理问题。

系统采用前后端分离架构：

- 后端：Spring Boot 3、Spring Security、JWT、MyBatis-Plus
- 前端：Vue 3、TypeScript、Element Plus、ECharts、Vite
- 数据库：MySQL 8
- 缓存：Redis 7

系统上线初始化时不包含业务演示数据，仅保留角色、菜单和权限基础配置。管理员账号通过环境变量在首次启动时初始化创建。

## 2. 需求分析

### 2.1 系统目标

系统目标是建设一个实验室资源统一管理平台，实现实验室资产、耗材、危化品和业务流程的数字化管理，提升资源流转效率、安全追溯能力和统计分析能力。

主要目标包括：

- 建立设备、耗材、危化品统一台账。
- 支持设备借用、归还、维修、校准等完整流程。
- 支持耗材入库、出库、库存预警和成本统计。
- 支持危化品入库、领用、归还、废液处理和安全追踪。
- 支持基于角色的权限控制，保障不同用户访问不同业务范围。
- 支持通知提醒、操作审计和报表导出。

## 3. 数据库总体设计

数据库名称：`lab_management_system`

数据库类型：`MySQL 8`

当前数据库主要包括以下类型对象：

- 基础表：用户、角色、菜单、实验室、资产台账等。
- 业务表：设备借用、维修、校准、耗材出入库、危化品使用等。
- 库存表：统一维护耗材和危化品批次库存。
- 通知与日志表：用户通知、系统日志、操作审计日志。
- 视图：用于快速查询借用、库存和危化品汇总状态。
- 存储过程：用于设备借用、耗材出库和月度报表生成。
- 触发器：用于自动维护设备状态、库存和日志。

## 9. 主要业务流程

### 9.1 设备借用与归还流程

1. 用户选择设备并填写借用用途、借用时间和应还时间。
2. 系统校验设备是否处于可用状态。
3. 借用成功后生成设备借用记录。
4. 系统将设备状态更新为借出。
5. 归还时管理员填写归还情况。
6. 系统记录实际归还时间，并恢复设备可用状态。
7. 逾期记录可向借用账号发送催还通知。

### 9.2 耗材入库与出库流程

1. 耗材管理员登记耗材入库批次、数量、单价和有效期。
2. 系统自动更新库存批次。
3. 用户申请耗材出库。
4. 系统校验库存是否充足。
5. 出库成功后扣减库存并记录出库金额。
6. 库存不足或临期时产生预警。

### 9.3 危化品使用流程

1. 危化品管理员维护危化品台账。
2. 用户提交危化品领用、归还或废液处理记录。
3. 系统根据动作类型更新库存。
4. 系统保留用途、项目、见证人和余量信息。
5. 后续可根据记录进行安全追踪和统计分析。

### 9.4 设备维修与校准流程

1. 用户或管理员提交设备维修记录。
2. 维修人员更新维修状态、费用和处理结果。
3. 维修完成后设备恢复可用或进入报废状态。
4. 校准人员创建校准任务并登记证书编号。
5. 校准完成后更新最近校准日期和下次校准日期。

## 10. 数据库设计特点

- 所有业务表统一包含 `create_time`、`update_time`、`is_deleted` 字段。
- 状态字段统一使用 `TINYINT` 存储。
- 重要编码字段使用唯一约束，例如设备编号、耗材编号、危化品编号。
- 通过外键约束保证用户、实验室、设备、耗材等数据关系一致。
- 通过触发器自动维护库存和设备状态。
- 通过视图提升常用业务查询效率。
- 初始化脚本不包含业务演示数据，适合生产环境部署。

## 11. 结论

本系统数据库设计围绕实验室资源全生命周期管理展开，覆盖用户权限、实验室基础信息、设备管理、耗材管理、危化品管理、库存管理、通知提醒和日志审计等核心业务。

整体结构满足实验室日常管理、业务追踪、安全审计和统计分析需求，具备进一步扩展审批流、盘点管理、附件管理和消息推送的基础。
